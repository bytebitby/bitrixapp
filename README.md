# Bitrix24 local app: activity для внешнего вебхука

Это локальное приложение для Битрикс24, которое регистрирует пользовательское activity в бизнес-процессах.  
Activity вызывает внешний вебхук и возвращает результат в блок `Дополнительно` как `RETURN_PROPERTIES`.

## Что умеет MVP

- регистрирует activity через `install.php`
- обязательно содержит `handler.php`
- открывает собственный интерфейс настройки через `placement.php`
- принимает URL вебхука в параметрах activity
- вызывает вебхук во время выполнения бизнес-процесса
- возвращает в бизнес-процесс:
  - `webhook_result` — полный ответ вебхука
  - `http_status` — HTTP статус ответа
  - `error_message` — текст ошибки, если вызов не удался
- пишет сервисные логи в файл

## Структура проекта

- `bootstrap.php` — общие функции, конфиг, логирование, REST-вызовы в Битрикс24
- `install.php` — установка и обновление activity в портале
- `handler.php` — выполнение activity и возврат результата в БП
- `placement.php` — UI для настройки activity внутри Битрикс24
- `.env.example` — пример серверной конфигурации

## Как это работает

1. Локальное приложение в Битрикс24 открывает `install.php`.
2. `install.php` вызывает `bizproc.activity.add`.
3. Activity регистрируется с `USE_SUBSCRIPTION=Y` и `USE_PLACEMENT=Y`.
4. Пользователь добавляет activity в бизнес-процесс.
5. В окне настройки activity открывается `placement.php`, где вводится URL вебхука.
6. Во время выполнения БП Битрикс24 вызывает `handler.php`.
7. `handler.php` отправляет POST-запрос во внешний вебхук.
8. Результат возвращается в Битрикс24 через `bizproc.event.send`.

## Какие данные отправляются во внешний вебхук

Приложение отправляет JSON:

```json
{
  "activity_id": "ID activity",
  "workflow_id": "ID бизнес-процесса",
  "document_id": ["массив идентификаторов документа"],
  "portal_domain": "portal.bitrix24.ru",
  "event_token": "token"
}
```

## Требования

- публично доступный сервер
- приложение должно быть доступно до добавления в Битрикс24
- для Bitrix24 cloud рекомендуется внешний URL по HTTPS
- PHP с расширением `curl`
- веб-сервер с корректным document root

## Важное по текущему серверу

На дату проверки `20.04.2026` сервер `89.169.154.151`:

- отвечает по `http://89.169.154.151/`
- отдает стандартную страницу Apache Ubuntu
- `http://89.169.154.151/install.php` возвращает `404`
- `http://89.169.154.151/handler.php` возвращает `404`

Это значит, что проект пока не развернут в веб-корне и сервер нужно подготовить отдельно.

## Настройка на сервере

### 1. Разместить проект

Рекомендуемая папка:

```bash
/var/www/html/bitrixapp
```

### 2. Подготовить переменные окружения

Создайте `.env` или задайте переменные окружения в конфиге веб-сервера:

```env
APP_BASE_URL=https://your-domain.example/bitrixapp
APP_LOG_DIR=/var/www/html/bitrixapp/var
APP_ACTIVITY_CODE=bytebit_webhook_activity
```

Если используете только IP без домена, приложение тоже сможет работать, но для Bitrix24 cloud лучше сразу поднять домен и HTTPS.

### 3. Настроить веб-сервер

Нужно, чтобы следующие URL были доступны извне:

- `https://your-domain.example/bitrixapp/install.php`
- `https://your-domain.example/bitrixapp/install.php`
- `https://your-domain.example/bitrixapp/handler.php`
- `https://your-domain.example/bitrixapp/placement.php`

### 4. Проверить PHP

Нужен модуль `curl`. Проверка:

```bash
php -m | grep curl
```

## Как добавить приложение в Битрикс24

1. Перейдите в `Приложения -> Разработчикам -> Интеграции`.
2. Создайте локальное приложение.
3. Укажите права минимум на `bizproc`.
4. В качестве пути первоначальной установки укажите:

```text
https://your-domain.example/bitrixapp/install.php
```

5. Сохраните приложение и откройте его.
6. `install.php` зарегистрирует activity и завершит установку через `BX24.installFinish()`.

## Как использовать activity

1. Откройте дизайнер бизнес-процесса.
2. Добавьте activity `Вызов внешнего вебхука`.
3. В настройках activity введите URL вебхука.
4. Сохраните шаблон БП.
5. После выполнения activity откройте раздел `Дополнительно`.
6. Используйте возвращаемые значения:
   - `webhook_result`
   - `http_status`
   - `error_message`

## Логи

По умолчанию приложение пишет логи в:

```text
var/app.log
```

На сервере путь лучше задавать через `APP_LOG_DIR`.

## Локальная разработка

Пример запуска встроенного PHP-сервера:

```bash
php -S 127.0.0.1:8080
```

После этого приложение будет доступно локально по:

```text
http://127.0.0.1:8080/install.php
```

Для реального подключения к Bitrix24 нужен внешний адрес, доступный из интернета.

## Деплой на сервер

Ниже базовый вариант через `scp`:

```bash
scp -r . root@89.169.154.151:/var/www/html/bitrixapp
```

После копирования на сервере нужно:

```bash
chown -R www-data:www-data /var/www/html/bitrixapp
mkdir -p /var/www/html/bitrixapp/var
```

И настроить nginx так, чтобы PHP обрабатывался внутри `/var/www/html/bitrixapp`.
